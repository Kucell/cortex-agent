# 代码提交工作流 (/commit)

1. **状态检查**:
    - 运行 `git status` 查看待提交的改动。
2. **代码自检与评估**:
    - 对当前改动进行最后的自我审查，确保没有多余的调试代码或不符合规范的改动。
3. **质量验证**:
    - 运行项目配置的类型检查工具。
    - 运行相关的单元测试，确保无回归问题。
4. **生成提交信息**:
    - 参考 `.agent/rules/commit-standards.md`。
    - 确定改动的 `type` 和 `scope`。
    - 根据项目习惯编写 `subject`。
5. **执行提交**:
    - 确认生成的提交信息无误。
    - 执行 `git commit -m "[提交信息]"`。
6. **后续操作**:
    - 确认是否需要推送到远程仓库 (`git push`)。
