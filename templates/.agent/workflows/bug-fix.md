# 缺陷修复工作流 (/bug-fix)

1. **Bug 确认与分类**:
    - **分类**: 类型错误、运行时错误、逻辑错误或架构设计问题。
    - **上下文收集**: 阅读错误堆栈，使用代码搜索工具 (grep/rg) 定位相关代码块。
2. **根因分析与定位**:
    - 追踪数据流或控制流以找到错误源头。
    - 使用 `code-evaluation` 技能评估是否存在生命周期、时机或分工问题。
    - 检查是否违反了项目定义的架构设计规则。
3. **复现与测试**:
    - **自动化复现**: 优先编写一个能触发该 Bug 的失败测试用例。
    - **手动复现**: 在本地环境中还原问题场景，观察其对系统的副作用。
4. **修复实施**:
    - **寻找最佳修复位**: 优先考虑在通用层级修复系统性问题，避免在叶子节点打补丁。
    - **规范化编码**: 确保类型安全，添加必要的健壮性检查，遵循 `.agent/rules/code-standards.md`。
5. **验证修复**:
    - 运行针对该 Bug 的测试用例，确保通过。
    - 运行全量测试套件和类型检查，确保没有引起回归。
6. **总结与预防**:
    - 确认回归测试已包含在提交中。
    - 如果该 Bug 揭示了某种共性问题，考虑更新相关的 Rule 或文档以提醒其他开发者。
