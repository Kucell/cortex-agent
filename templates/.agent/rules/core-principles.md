# 核心开发原则 (模板)

## 回复与输出
- **精简高效**：优先提供可执行的代码解决方案，避免过度冗余的解释。
- **结果导向**：确保输出的代码逻辑完整，符合项目标准。
- **可视化支持**：对于复杂逻辑，建议使用 Mermaid 图表进行辅助说明。
- **语言规范**：根据项目约定（如：中文/英文）生成文档和注释。

## 任务执行流程
- **状态确认**：在开始新任务前，务必查阅当前的任务进度和计划。
- **方案评估**：在动手编码前，先对实现方案进行快速评估，考虑其对现有系统的影响。
- **质量验证**：完成开发后，应运行项目定义的测试集和类型检查工具。
- **进度更新**：任务完成后，及时更新相关的任务追踪文档。

## 开发实践
- **工具链使用**：熟练使用项目配置的 CLI 工具、包管理器和调试工具。
- **破坏性改动预防**：在执行风险较高的操作（如删除、大规模重构）前，务必做好备份或确认。
- **渐进式改进**：遵循“先工作，后重构”的原则，确保每一步改动都是可回滚且可验证的。

## 鲁棒性与规范
- **防御式编程**：处理边界条件、异常流和资源释放。
- **回归测试**：修复 Bug 时应同步添加/更新相关的测试用例以防止回归。
- **一致性**：保持代码风格、目录结构和命名习惯与现有项目高度一致。
