# 代码实现评估技能 (SKILL)

此技能提供了一套结构化方法，用于评估代码实现方案的合理性、可靠性及性能影响。

## 评估维度

### 1. 实施时机 (Timing & Lifecycle)
- **阶段确认**: 代码应在系统的哪个生命周期阶段运行？
- **依赖依赖**: 执行前必须满足哪些前置条件？
- **异步处理**: 涉及异步操作时，是否存在潜在的竞态条件 (Race Conditions)？是否需要重试或等待机制？

### 2. 位置选择 (Location & Context)
- **层级归属**: 应在底层基础设施层、中间件层还是顶层业务应用层？
- **模块职责**: 此逻辑是否与其所在的类或函数职责相符？
- **可见性**: 是否应该暴露为公开 API 或保持为内部实现细节？

### 3. 实现方案 (Implementation Strategy)
- **简洁性 vs. 扩展性**: 方案是否过于复杂？是否存在更简单的替代方案？
- **性能预期**: 在高频调用或大数据量场景下，是否存在性能瓶颈？
- **异常边界**: 方案如何处理错误、空值或非法输入？

---

## 推荐实践：可靠性增强

在涉及不确定性环境（如：外部服务调用、并发文件系统访问、异步 UI 渲染等）时，建议如下模式：

- **重试机制**: 对于偶发性失败，应引入带有退避策略 (Backoff) 的重试逻辑。
- **状态验证**: 执行核心逻辑前后，验证系统的状态一致性。
- **资源清理**: 确保任何分配的资源（监听器、内存挂载、文件句柄）都被正确释放。

## 使用场景

在以下情况，建议先进行全方位的评估分析再动手编码：
1. 涉及系统核心逻辑的修改。
2. 修复难以复现的随机 Bug。
3. 引入新的重型依赖或三方库。
4. 进行大规模的重构。

## 评估输出格式

1. **现状分析**: 简述代码目前存在的问题或需求背景。
2. **三维度分析**:
   - **时机**: [分析结论]
   - **位置**: [分析结论]
   - **方案**: [分析结论]
3. **最终决策**: 确定的实施步骤。
