# 架构审计技能 (SKILL)

此技能用于评估代码变动或新设计方案是否符合项目的核心架构原则。

## 执行指南

对当前代码状态或提议的设计变动执行以下维度的审计：

### 1. 职责边界检查
- **[维度名称]**: 逻辑是否被放置在了错误的层级（例如：在 UI 组件中编写复杂的业务逻辑）？
- **[维度名称]**: 是否违反了单一职责原则 (SRP)？

### 2. 依赖关系检查
- 是否引入了不合理的循环依赖？
- 模块之间的耦合度是否过高？
- 是否违反了定义的依赖方向（例如：核心逻辑不应依赖于具体的基础设施实现）？

### 3. 可维护性与扩展性检查
- 是否预留了足够的扩展接口？
- 是否存在过度设计 (Over-engineering) 或设计不足 (Under-engineering)？

### 4. 平台与环境检查
- 逻辑是否具备良好的可移植性？
- 是否正确隔离了平台特有的 API (如：Browser API, Node.js API)？

---

## 方案对比审计 (Comparative Audit)

当需要对多个设计方案进行对比分析时：

1. **核心原则对齐**: 评估各方案对项目核心原则（如：高性能、高可用、低耦合等）的遵循程度。
2. **风险评估**: 识别各方案可能带来的技术债、性能开销或安全性风险。
3. **权衡分析**: 明确每个方案的优势 (Pros) 和劣势 (Cons)。
