# 项目内部插件 (Internal Plugins)

## 🎯 功能定位

此目录用于存放项目**内部的**、用于扩展 Agent 核心能力的“插件化”模块。

这些模块不是为了跨项目分发，而是作为当前项目专属的核心能力扩展。

## 🔎 与“可分发插件 (Distributable Plugins)”的区别

需要明确区分本目录的用途与 Claude Code 等平台的“可分发插件”概念：

-   **内部插件 (本目录)**:
    -   **范围**: 仅限当前项目使用。
    -   **集成方式**: 直接作为源码存在于项目结构中，通过 `.agent/` 目录被 Agent 直接加载和使用。
    -   **用途**: 集成项目特定的工具、连接内部数据库或 API (例如，实现一个 MCP 服务)、定义不适合放在 `skills` 中的复杂或共享逻辑。

-   **可分发插件 (Claude Code 定义)**:
    -   **范围**: 可跨项目、跨团队共享和复用。
    -   **集成方式**: 通过“Marketplace”进行发现、安装和版本管理。其本身是一个包含 `plugin.json` 清单文件的独立包。
    -   **用途**: 发布通用的、标准化的能力包（例如，一个用于 Git 操作的完整技能和命令包）。

## 📝 结构说明

一个内部插件可以是一个包含配置文件和相关脚本的子目录，其具体结构相对灵活，旨在被项目内的其他 `skills` 或 `workflows` 调用。
