# Git 提交规范

遵循 [Conventional Commits 1.0.0](https://www.conventionalcommits.org/zh-hans/v1.0.0/) 规范。

## 消息格式

```
<type>(<scope>): <subject>

[body]

[footer]
```

- **type**：见下方类型表，全小写。
- **scope**（可选）：受影响的模块或目录，如 `auth`、`api`、`ui`。
- **subject**：简洁描述，不超过 72 字符，**不加句号**。
- **body**（可选）：详细说明改动原因与内容，每行不超过 100 字符。
- **footer**（可选）：关联 Issue（`Closes #123`）或声明破坏性变更（`BREAKING CHANGE:`）。

## 类型（Type）

| type | 用途 |
|------|------|
| `feat` | 新功能 |
| `fix` | Bug 修复 |
| `docs` | 仅文档变更 |
| `style` | 格式调整（不影响逻辑：空格、分号等） |
| `refactor` | 重构（非 bug 修复、非新功能） |
| `perf` | 性能优化 |
| `test` | 添加或修正测试 |
| `build` | 构建系统或依赖变更 |
| `ci` | CI/CD 配置变更 |
| `chore` | 其他维护性变更 |
| `revert` | 回退某次提交 |

## 破坏性变更（BREAKING CHANGE）

在 footer 中声明：

```
BREAKING CHANGE: <描述兼容性被破坏的地方>
```

或在 type/scope 后加 `!`：`feat(api)!: 重命名用户端点`

## 语言规范

- **subject 和 body 的语言必须与项目配置一致**。
- 读取 `.agent/rules/tech-stack.md` 或 `AGENTS.md` 中的语言偏好；若无明确配置，默认使用**中文**。
- 同一条提交信息中不得混用中英文。

## 禁止事项

以下内容**严禁**出现在任何提交信息的任何部分（type、subject、body、footer）：

- `Co-authored-by: Claude`
- `Co-authored-by: AI`
- `Generated by AI` / `AI-generated` / `AI assisted`
- `Generated with Claude` / `Powered by Claude`
- 任何形式的 AI 工具名称作为署名或协作者

> 提交信息代表开发者本人的工作，AI 仅作为辅助工具，不得以任何形式出现在提交记录中。

## 示例

```
feat(auth): 新增 OAuth2 登录支持

支持 GitHub 和 Google 两种第三方登录方式。
后端新增 /auth/callback 路由，前端新增登录按钮组件。

Closes #42
```

```
fix(api): 修复用户列表分页越界问题

当 page 参数超出总页数时，接口返回空数组而非 500 错误。
```

```
refactor(core)!: 移除旧版配置加载器

BREAKING CHANGE: loadConfig() 已被移除，请改用 loadConfigAsync()
```
